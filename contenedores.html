Manejo de Contenedores

Tecnologías para empaquetar, distribuir y ejecutar aplicaciones en entornos aislados

¿Qué son los contenedores?

Los contenedores son unidades ejecutables de software en las que se empaqueta el código de la aplicación junto con sus bibliotecas y dependencias, de manera que se puede ejecutar en cualquier entorno informático.

Los contenedores virtualizan el sistema operativo, a diferencia de las máquinas virtuales que virtualizan el hardware. Esto los hace más ligeros y eficientes, ya que comparten el kernel del sistema operativo host.

Dato interesante Ahora
¿Sabías que Docker, la plataforma de contenedores más popular, fue lanzada en 2013 y revolucionó la forma en que se desarrollan y despliegan aplicaciones?

Ventajas de los contenedores

Portabilidad

Los contenedores encapsulan todas las dependencias, lo que permite que las aplicaciones se ejecuten de manera consistente en diferentes entornos, desde el desarrollo hasta la producción.

Eficiencia

Los contenedores comparten el kernel del sistema operativo host, lo que los hace más ligeros y rápidos que las máquinas virtuales tradicionales.

Escalabilidad

Los contenedores facilitan la escalabilidad horizontal, permitiendo aumentar o disminuir rápidamente el número de instancias de una aplicación según la demanda.

Aislamiento

Cada contenedor funciona de manera aislada, lo que mejora la seguridad y evita conflictos entre aplicaciones que comparten el mismo host.

Docker

La plataforma de contenedores más popular que permite a los desarrolladores empaquetar aplicaciones en contenedores. Docker proporciona un conjunto completo de herramientas para crear, distribuir y ejecutar contenedores.

Visitar
Kubernetes

Sistema de orquestación de contenedores de código abierto que automatiza la implementación, el escalado y la gestión de aplicaciones en contenedores. Kubernetes facilita la administración de clústeres de contenedores a gran escala.

Visitar
containerd

Un runtime de contenedores de nivel industrial que gestiona el ciclo de vida completo de los contenedores, desde la transferencia de imágenes hasta la ejecución, supervisión y gestión de red.

Visitar
Podman

Una alternativa a Docker que no requiere un daemon en ejecución. Podman puede ejecutar contenedores como usuario no privilegiado y ofrece compatibilidad con la API de Docker.

Conceptos clave de contenedores

Glosario de términos
Aquí encontrarás definiciones de los términos más importantes relacionados con los contenedores.
Imagen de contenedor

Plantilla de solo lectura con instrucciones para crear un contenedor.

Dockerfile

Archivo de texto que contiene instrucciones para construir una imagen de Docker.

Orquestación

Proceso de automatizar el despliegue, gestión, escalado y networking de contenedores.

Pod

En Kubernetes, un grupo de uno o más contenedores con almacenamiento y red compartidos.

Volumen

Mecanismo para persistir datos generados y utilizados por contenedores.

Registry

Repositorio para almacenar y distribuir imágenes de contenedores.

Namespace

Mecanismo para aislar grupos de recursos dentro de un clúster.

Arquitectura de contenedores

Los contenedores se basan en varias tecnologías del kernel de Linux, como namespaces y cgroups, que proporcionan aislamiento y control de recursos.

  • Namespaces: Proporcionan aislamiento para procesos, redes, sistemas de archivos, etc.
  • Control Groups (cgroups): Limitan y aíslan el uso de recursos como CPU, memoria y E/S.
  • Union File Systems: Permiten que las capas de imágenes sean apiladas y vistas como un sistema de archivos único.

Flujo de trabajo con contenedores

1
Crear un Dockerfile

Define las instrucciones para construir la imagen del contenedor, incluyendo el sistema operativo base, dependencias, archivos de la aplicación y configuraciones.

FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
2
Construir la imagen

Utiliza el Dockerfile para crear una imagen que contiene todo lo necesario para ejecutar la aplicación.

docker build -t mi-aplicacion:1.0 .
3
Ejecutar el contenedor

Inicia un contenedor basado en la imagen construida.

docker run -p 3000:3000 mi-aplicacion:1.0
4
Distribuir la imagen

Sube la imagen a un registro de contenedores para compartirla con otros o desplegarla en diferentes entornos.

docker push mi-registro/mi-aplicacion:1.0
5
Orquestar contenedores

Utiliza herramientas como Kubernetes o Docker Swarm para gestionar múltiples contenedores, escalado, balanceo de carga y recuperación ante fallos.
